What OpenClaw Actually Needs from a VPS

OpenClaw is an autonomous operator agent built on Node.js. Before comparing providers, it's worth understanding what the platform actually consumes at runtime so you don't overpay for resources you don't need — or worse, underspend and watch your agent OOM crash at 3 AM.

OpenClaw's resource profile breaks down into a few distinct categories:

  • Node.js runtime: Node.js 18+ is required. At idle with a single active session, it consumes ~150–300 MB RSS. Under active reasoning or tool execution, it spikes to 500 MB–1 GB briefly before settling.
  • LLM API calls: OpenClaw doesn't run local models by default — it calls external LLM APIs (OpenAI, Claude, Groq, etc.). This means CPU is less critical than network throughput. Your VPS's outbound bandwidth matters more than its core count.
  • Persistent sessions and memory: OpenClaw maintains long-running session state. More RAM lets it hold more conversation context without paging to disk.
  • Storage: Logs, session files, skill data, and optional local caching. Not demanding — 10–20 GB is plenty for a single-agent deployment.
  • Network: Outbound API calls to LLM providers, plus inbound connectivity for Telegram/webhooks if you're using OpenClaw's messaging integrations. Static IP and IPv6 support are valuable here.
Minimum viable VPS for a single OpenClaw agent: 2 GB RAM, 2 vCPU, 20 GB storage. Recommended: 4 GB RAM, 2+ vCPU, NVMe storage. Production multi-skill deployments: 8 GB RAM minimum.

Ubuntu Setup for OpenClaw

Once you've chosen your VPS, here's the streamlined path to get OpenClaw running on Ubuntu 22.04 LTS or 24.04 LTS.

Step 1: Create and Access Your VPS

For Hetzner: use the Cloud Console or the hcloud CLI to create a new server. Select Ubuntu 22.04 LTS as the image. For Vultr: use the control panel — Ubuntu 22.04 LTS is a one-click option. For Contabo: select Ubuntu 22.04 during ordering.

Connect via SSH using the root credentials sent to your email:

ssh root@YOUR_VPS_IP

Step 2: Create a Non-Root User

Never run OpenClaw as root. Create a dedicated user:

adduser openclau
usermod -aG sudo openclau
su - openclau

Step 3: Install Node.js 20 LTS

OpenClaw requires Node.js 18 or higher. Use NodeSource for the LTS release:

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node --version  # Should print v20.x.x
npm --version

Step 4: Install OpenClaw

Clone or install OpenClaw per the official docs. Typically:

npm install -g openclaw
openclaw gateway start

Configure your gateway with the environment variables for your LLM provider (e.g., OPENAI_API_KEY), messaging platform tokens, and storage path. See the full 24/7 setup guide for systemd service configuration and auto-restart.

Step 5: Run as a Systemd Service

Create a systemd service so OpenClaw survives reboots and restarts on failure:

sudo nano /etc/systemd/system/openclaw.service
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
User=openclau
WorkingDirectory=/home/openclau
ExecStart=/usr/bin/openclaw gateway start
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
sudo systemctl enable openclaw
sudo systemctl start openclaw
sudo systemctl status openclaw

Frequently Asked Questions

Can I run OpenClaw on a 2 GB RAM VPS?

Yes, technically. A single OpenClaw agent on Node.js with 2 GB RAM will run without swapping under light load. But headroom is minimal — any spike in tool execution, concurrent sessions, or logging will push you into swap, which degrades response times. 4 GB is the practical minimum. 8 GB+ is recommended for anything beyond experimentation.

Does OpenClaw need a dedicated IP address?

Yes, if you're using Telegram webhooks or any inbound integration. OpenClaw's messaging integrations require a publicly reachable endpoint. A static IP from your VPS provider is essential — dynamic IPs from residential connections or shared hosting will break webhook delivery. All three providers compared here offer static IPs by default on VPS instances.

Is NVMe important for OpenClaw?

Not as critical as RAM, but it helps. OpenClaw reads and writes session logs, skill data, and optional caches. NVMe reduces log rotation latency and speeds up any local model inference you add later (via Ollama, for example). For pure API-calling OpenClaw deployments, a good SSD is sufficient. For anything running local models or large skill datasets, NVMe is worth the small price premium.

How do I monitor OpenClaw on a VPS?

Use pm2 (Process Manager 2) or systemd for process monitoring. Pair it with UptimeRobot or Uptime Kuma for external uptime checks. For resource monitoring, htop or NetData on the VPS itself. The systemd service configuration above handles auto-restart on crash — pair that with a free uptime monitor and you'll get alerted before your agent goes silent.

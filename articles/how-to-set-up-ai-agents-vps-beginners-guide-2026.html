How to Set Up AI Agents on a VPS — Beginner's Guide 2026

📅 March 25, 2026 ⏱️ 12 min read 🏷️ VPS Hosting, AI Agents

You don't need a computer science degree to run AI agents on a server. With the right VPS and a few terminal commands, you can deploy autonomous agents that work 24/7 — handling research, content creation, coding tasks, and more — while you sleep.

This guide walks beginners through the complete process: choosing a VPS, installing the necessary software, and getting your first AI agent running from scratch. No prior server experience required.

What You Need Before Starting

  • A VPS server — Starting at $5/month. Recommended: Hetzner, Contabo, or DigitalOcean.
  • A domain name (optional but recommended) — Around $10–15/year from Namecheap or Cloudflare.
  • SSH client — Built into Mac/Linux terminal. Windows users: PuTTY or Windows Terminal.
  • An AI provider API key — OpenAI, Anthropic, or MiniMax. Most offer free credits to start.

💡 The VPS Approach vs. Cloud Platforms

Platforms like ChatGPT Plus ($20/month) or Claude Pro give you access to AI, but the agent stops when you close the app. A VPS-hosted agent runs autonomously — no human needed, no session time limits.

For AI agents, you want a VPS with enough RAM to handle model inference. Here's what we recommend at minimum:

Step 2 — Set Up Your Server

2.1 Connect via SSH

After ordering your VPS, you'll receive an IP address and root password. Open your terminal and connect:

ssh root@YOUR_VPS_IP

On first connection, your terminal will ask to confirm the host key — type yes and press Enter.

2.2 Update Your Server

Always start with a full system update:

apt update && apt upgrade -y

2.3 Install Required Software

AI agent frameworks like OpenClaw require Node.js (version 18+) and npm. Install them:

# Install Node.js 22.x
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt-get install -y nodejs

# Verify installation
node --version   # Should show v22.x.x
npm --version    # Should show 10.x.x

# Install pm2 (keeps your agent alive after server restarts)
npm install -g pm2

2.4 Configure the Firewall

Allow only the ports you need:

ufw allow ssh
ufw allow 80/tcp   # HTTP
ufw allow 443/tcp  # HTTPS
ufw enable
⚠️ Warning: Never run ufw enable if you're connecting via SSH without port 22 allowed. You will lock yourself out of your server.

Step 3 — Install and Configure OpenClaw

OpenClaw is a free, open-source framework for running autonomous AI agents on a VPS. It handles the heavy lifting — agent orchestration, memory, tools, and channel integrations (Telegram, WhatsApp, Discord).

# Install OpenClaw globally
npm install -g openclaw

# Verify installation
openclaw --version

# Run the setup wizard (interactive)
openclaw setup

The setup wizard will ask for:

  • AI Provider — Choose your API key source (OpenAI, Anthropic, MiniMax, etc.)
  • Channel integrations — Telegram bot token, WhatsApp if needed
  • Memory storage — Local file storage is fine for single-agent setups

3.1 Start OpenClaw as a Background Service

Use PM2 so your agent survives server reboots:

# Start OpenClaw in the background
pm2 start openclaw --name "openclaw-agent" --interpreter node

# Save the PM2 process list so it restarts on reboot
pm2 save

# Set up startup script
pm2 startup

The pm2 startup command will output a system command — copy and run it as root to enable automatic restarts.

Step 4 — Point a Domain to Your Agent (Optional)

If you want a web dashboard or webhook access, point a subdomain at your VPS:

  1. In your domain registrar (Cloudflare, Namecheap), add an A record:
    • Host: agent (or bot)
    • Value: YOUR_VPS_IP
    • TTL: Auto
  2. Wait 5–30 minutes for DNS propagation
  3. Your agent dashboard will be accessible at https://agent.yourdomain.com

Step 5 — Verify Everything Is Running

Check the status of your agent:

# Check PM2 status
pm2 status

# View real-time logs
pm2 logs openclaw-agent --lines 50

# Check if the gateway is accessible
curl http://localhost:18789/__openclaw__/api/v1/status

You should see your agent's status, uptime, and configured channels.

Common Beginner Mistakes to Avoid

✅ Do This

  • Use PM2 to keep your agent alive
  • Set up automated backups of your config
  • Monitor RAM usage with htop
  • Keep your VPS OS and packages updated
  • Use environment variables for API keys, never hardcode them

❌ Avoid This

  • Don't run agents as root unnecessarily
  • Don't skip firewall configuration
  • Don't store API keys in config files committed to git
  • Don't use very cheap VPS with swap-only storage (terrible I/O)
